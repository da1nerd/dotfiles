# Vim slim-down implementation plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Shrink `vimrc.symlink` from ~330 lines to ~45 with two plugins (base16-vim, vim-fugitive), drop dead snippets and the `vim=nvim` alias lie.

**Architecture:** Personal dotfiles repo. One main config file (`vimrc.symlink`, symlinked to `~/.vimrc`), two plugins via vim-plug. No test suite — verification is running vim and observing behavior.

**Tech Stack:** vim (plain, not nvim), vim-plug, bash, git.

**Spec:** `docs/superpowers/specs/2026-04-20-vim-slim-design.md`

### Task 1: Rewrite `vimrc.symlink`

**Files:**
- Modify: `vimrc.symlink` (full rewrite)

**Why this is one task:** the file is small enough to replace whole-cloth. Trying to land incremental edits to a 330-line file would be more error-prone than a clean rewrite.

- [ ] **Step 1: Confirm the current file**

Run: `wc -l /home/joel/.dotfiles/vimrc.symlink`
Expected: a line count in the 320–340 range. If you see ~45 already, the rewrite has happened — stop and ask.

- [ ] **Step 2: Write the new `vimrc.symlink`**

Replace the entire contents of `/home/joel/.dotfiles/vimrc.symlink` with:

```vim
" Minimal vim config — for git commits, quick edits, config tweaks.
" Full IDE work happens in VS Code or Claude Code.

set nocompatible
set nomodeline
set autoread
set backspace=indent,eol,start
set clipboard^=unnamed,unnamedplus

let mapleader = ','

" Undo across sessions
set undofile
set undodir=~/.vim-tmp/undodir

" Indentation (tabs, 4-wide)
set noexpandtab
set smarttab
set tabstop=4
set softtabstop=4
set shiftwidth=4

" UI
set number
set laststatus=2
set showcmd

" Search
set ignorecase
set smartcase
set hlsearch
set incsearch

" jj → ESC
inoremap jj <ESC>

" Plugins
call plug#begin('~/.vim/plugged')
Plug 'chriskempson/base16-vim'
Plug 'tpope/vim-fugitive'
call plug#end()

" Colorscheme (from shell $THEME / $BACKGROUND)
syntax on
let base16colorspace=256
if !empty($BACKGROUND)
    execute "set background=" . $BACKGROUND
endif
if !empty($THEME)
    silent! execute "colorscheme " . $THEME
endif

" Fugitive
nmap <silent> <leader>gs :Git<cr>
nmap <leader>ge :Gedit<cr>
nmap <silent> <leader>gr :Gread<cr>
nmap <silent> <leader>gb :Git blame<cr>
```

Use the `Write` tool (this is a full rewrite, not an edit).

- [ ] **Step 3: Verify vimscript parses**

Run: `vim -u /home/joel/.dotfiles/vimrc.symlink -c 'q' 2>&1 | head`
Expected: empty output. The `silent!` prefix on `colorscheme` suppresses the first-run E185 "cannot find color scheme" error that would otherwise appear before PlugInstall runs.

If you see errors like `E492: Not an editor command` or `E488: Trailing characters`, the rewrite has a syntax bug — re-check the file contents byte-for-byte against Step 2.

- [ ] **Step 4: Install plugins fresh**

First, clear any existing plugin install to make sure PlugInstall actually runs:

```bash
rm -rf ~/.vim/plugged
```

Then:

```bash
vim +PlugInstall +qall
```

Expected: vim opens, downloads `base16-vim` and `vim-fugitive` into `~/.vim/plugged/`, then quits. Takes a few seconds. No interactive prompts.

Verify:

```bash
ls ~/.vim/plugged
```

Expected output: two directories — `base16-vim` and `vim-fugitive`.

- [ ] **Step 5: Smoke test the config**

Create a throwaway file, open it, check a few things:

```bash
echo "hello" > /tmp/vim-smoke-test.txt
vim /tmp/vim-smoke-test.txt
```

Inside vim, verify:
1. Line numbers show on the left (`:set number?` → `number`)
2. `:Git` opens a fugitive status buffer (no error like `E492`)
3. In insert mode, typing `jj` exits to normal mode
4. `:echo $THEME` shows the current shell theme (probably `base16-atelier-lakeside`)

Quit without saving: `:q!`. Clean up: `rm /tmp/vim-smoke-test.txt`.

If any check fails, fix the vimrc and retry.

- [ ] **Step 6: Commit**

```bash
git -C /home/joel/.dotfiles add vimrc.symlink
git -C /home/joel/.dotfiles commit -m "$(cat <<'EOF'
phase 2a: slim vimrc to ~45 lines

Drop NERDTree, CtrlP, airline, syntastic, devicons, nerdtree-git-plugin,
airline-themes. Drop custom functions (WinMove, ApplyLocalSettings,
Smart_TabComplete, HiInterestingWord, HtmlUnEscape, TrimWhiteSpace).

Keep: base16-vim, vim-fugitive. Modernize Gstatus/Gblame to Git/Git blame.

Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>
EOF
)"
```

Verify: `git -C /home/joel/.dotfiles log --oneline -1` shows the new commit.

## Rollback

If something goes sideways after landing:

```bash
git -C /home/joel/.dotfiles log --oneline | grep 'phase 2a'
git -C /home/joel/.dotfiles revert <commit-sha>
```

Both phase 2a commits can be reverted independently. Plugin dir (`~/.vim/plugged`) will still contain the old or new plugins — to match a reverted vimrc, run `vim +PlugInstall +PlugClean! +qall`.
