# Tmux slim-down implementation plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace broken tmux 2.3-era config with a minimal ~20-line tmux.conf, delete the dead theme files and session helpers.

**Architecture:** Personal dotfiles repo. One tmux config file (`tmux/tmux.conf.symlink`, symlinked to `~/.tmux.conf`). No plugins, no theme, no helpers — match the vim slim-down precedent.

**Tech Stack:** tmux 3.2a, bash, git. No test suite — verification is `tmux show-options` and grep.

**Spec:** `docs/superpowers/specs/2026-04-20-tmux-slim-design.md`

### Task 1: Rewrite `tmux/tmux.conf.symlink`

**Files:**
- Modify: `/home/joel/.dotfiles/tmux/tmux.conf.symlink` (full rewrite)

- [ ] **Step 1: Confirm the current file**

Run: `wc -l /home/joel/.dotfiles/tmux/tmux.conf.symlink`
Expected: line count in the 110-120 range. If already ~20 lines, the rewrite has happened — stop and ask.

- [ ] **Step 2: Write the new `tmux.conf.symlink`**

Use the `Write` tool to replace the entire contents of `/home/joel/.dotfiles/tmux/tmux.conf.symlink` with:

```tmux
# Minimal tmux config — defaults are mostly fine, these are worth setting.

# Ctrl-a prefix (ergonomics: C-b is awkward)
set -g prefix C-a
unbind C-b
bind C-a send-prefix

# Proper 256-color support
set -g default-terminal "screen-256color"

# Mouse on — pane resize, window switch, selection
set -g mouse on

# 1-indexed windows and panes (keyboard reach)
set -g base-index 1
setw -g pane-base-index 1
set -g renumber-windows on

# vi copy-mode (matches vim muscle memory)
setw -g mode-keys vi

# Sensible scrollback
set -g history-limit 10000

# Shorter escape for vim responsiveness
set -sg escape-time 10
```

- [ ] **Step 3: Load the config into a throwaway tmux server and verify it parses**

First, kill any running tmux server so we're testing a clean load:

```bash
tmux kill-server 2>/dev/null; true
```

Start a detached session using the new config directly (not via the symlink, so we test the repo file):

```bash
tmux -f /home/joel/.dotfiles/tmux/tmux.conf.symlink new-session -d -s smoke 2>&1
```

Expected: empty output. Any output is a parse error and must be fixed before proceeding.

- [ ] **Step 4: Verify each option landed correctly**

All five checks should pass against the running `smoke` session:

```bash
tmux show-options -g prefix
tmux show-options -g base-index
tmux show-options -g mouse
tmux show-window-options -g mode-keys
tmux show-options -g history-limit
```

Expected outputs:
- `prefix C-a`
- `base-index 1`
- `mouse on`
- `mode-keys vi`
- `history-limit 10000`

If any value is wrong, fix the config and restart the server (`tmux kill-server` then repeat Step 3).

- [ ] **Step 5: Clean up the test server**

```bash
tmux kill-server
```

No session survives to pollute the user's environment.

- [ ] **Step 6: Commit**

```bash
git -C /home/joel/.dotfiles add tmux/tmux.conf.symlink
git -C /home/joel/.dotfiles commit -m "$(cat <<'EOF'
phase 2b: slim tmux.conf to ~20 lines

Drop tmux 2.3-era syntax (bind -t vi-copy, window movement rebinds, fullscreen
layout, etc.) and the source of the broken theme.sh. Keep: Ctrl-a prefix,
mouse, vi copy-mode, 1-indexed windows/panes, sensible scrollback.

Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>
EOF
)"
```

Verify: `git -C /home/joel/.dotfiles log --oneline -1` shows the new commit. `git diff HEAD^ --stat tmux/` shows only `tmux.conf.symlink` modified, ~100 lines removed, ~20 added.

## Rollback

If something goes wrong after landing:

```bash
git -C /home/joel/.dotfiles log --oneline | grep 'phase 2b'

```

Both phase 2b commits can be reverted independently. Task 2's deletions restore on revert. Task 1's rewrite restores the old 115-line config on revert (which is still broken on modern tmux, but that's the pre-phase-2b state).
